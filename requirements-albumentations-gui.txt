# Optional Albumentations add-on for the desktop GUI environment.
#
# Install this after the main environment so pip does not pull opencv-python-headless: (If you get opencv assocated crashes; uninstall all opencv versions and then reinstall it)
#   pip install -r requirements.txt
#   pip install --no-deps -r requirements-albumentations-gui.txt
#
# This project keeps GUI-enabled OpenCV pinned in requirements.txt because the application uses
# cv2.imshow(), cv2.namedWindow(), and cv2.selectROI(). Installing the PyPI albumentations package
# with normal dependency resolution would add opencv-python-headless and can break the aforementioned features.
# This file therefore lists the needed transitive packages explicitly so "--no-deps" works in a fresh
# environment after the base IntegraPose install. 
# Do not run plain `pip install -r requirements-albumentations-gui.txt`, because that would still allow
# `albumentations` to add `opencv-python-headless`.
#
# Pin the Albumentations stack explicitly. IntegraPose does not use pydantic directly; it is listed
# here only because albumentations==1.4.21 declares pydantic>=2.7.0 and the --no-deps install flow
# requires us to include that transitive dependency ourselves.
